<h2>Step 1 ‚Äî What is Agile?</h2>
<p><strong>Agile</strong> is an <em>iterative &amp; incremental</em> way of working focused on <strong>value</strong>, <strong>fast feedback</strong>, and <strong>collaboration</strong>. Work is <strong>sliced</strong> into short <strong>Sprints</strong>, each producing a <strong>working Increment</strong>.</p>

<h4>Agile Manifesto: 4 values</h4>
<ul>
  <li><strong>Individuals &amp; interactions</strong> over processes &amp; tools.</li>
  <li><strong>Working product</strong> over comprehensive documentation.</li>
  <li><strong>Customer collaboration</strong> over contract negotiation.</li>
  <li><strong>Responding to change</strong> over following a plan.</li>
</ul>

:::note Important
‚Äú<em>Over</em>‚Äù means <strong>prioritize</strong>, not eliminate the right-hand items.  
:::

<h2>Step 2 ‚Äî When to use Agile/Hybrid</h2>

<h3>Use Agile when</h3>
<ul>
  <li>High volatility / high <strong>discovery</strong>.</li>
  <li>Need <strong>early time-to-value</strong> (MVP).</li>
  <li>High market risk ‚Üí learn fast from feedback.</li>
  <li><strong>Self-managing team</strong> &amp; engaged stakeholders.</li>
  <li>Work can be <strong>sliced</strong> into valuable increments.</li>
</ul>

<h3>Prefer Predictive/Hybrid when</h3>
<ul>
  <li><strong>Heavy compliance/safety</strong> environments.</li>
  <li><strong>Fixed scope/price</strong> contracts with limited change.</li>
  <li>Long/complex dependencies hinder regular increments.</li>
</ul>

<h3>Quick decision matrix</h3>

| Factor | Low | Medium | High |
|---|---|---|---|
| Requirement volatility | Predictive | Hybrid | Agile |
| Technical complexity | Predictive | Hybrid | Agile/Spikes |
| Compliance/Safety | Agile/Hybrid | Hybrid | Predictive/Hybrid |
| Early value need | Predictive | Hybrid | Agile |
| Increment slice-ability | Predictive | Hybrid | Agile |

<h2>Step 3 ‚Äî Agile vs Predictive vs Hybrid</h2>

<h4>Predictive (Waterfall)</h4>
<ul>
  <li><strong>Sequential</strong> phases and up-front detailed planning.</li>
  <li>Best when requirements are <strong>stable</strong>.</li>
</ul>

<h4>Agile</h4>
<ul>
  <li><strong>Iterative &amp; incremental</strong> Sprints; each delivers a <strong>usable Increment</strong>.</li>
</ul>

<h4>Hybrid</h4>
<ul>
  <li><strong>Combine</strong> Predictive for <strong>contracts/compliance</strong> and Agile for <strong>development/R&amp;D</strong>.</li>
</ul>

| Aspect | Agile | Predictive (Waterfall) | Hybrid |
|---|---|---|---|
| Approach | Iterative &amp; incremental | Sequential, up-front plan | Contextual mix |
| Scope | Evolves | Fixed early | Partly fixed |
| Value | Early via increments | End of project | Milestones + increments |
| Risk mgmt | Continuous, empirical | Front-loaded | Combined |
| Documentation | ‚ÄúJust enough‚Äù | Heavy | As required |
| Contracts | T&amp;M, value-based | Fixed scope/price | Milestones + increments |

1) Treating the Manifesto as ‚Äúno docs/process‚Äù.

3) Mixing up <strong>iterative</strong> vs <strong>incremental</strong>.  
4) Assuming Scrum is mandatory everywhere; forgetting <strong>tailoring</strong>.  
5) Ignoring continuous <strong>stakeholder engagement</strong>.  
:::
